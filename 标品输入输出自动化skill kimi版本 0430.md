
### 一、系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  节点0: 开始  │ ←── │  任意节点进入  │     │  支持断点续做 │
└──────┬──────┘     └─────────────┘     └─────────────┘
       ↓
┌─────────────────────────────────────────────────────────┐
│ 节点1: 上传文件（人工操作）                                │
│   └─ 支持格式: .pptx / .pdf / .jpg / .png                │
│   └─ 多文件批量上传                                      │
└────────────────────┬────────────────────────────────────┘
                     ↓ [确认]
┌─────────────────────────────────────────────────────────┐
│ 节点2: 数据提取（自动提取 → 人工确认/修改）                 │
│   └─ AI识别PPT/图片中的产品信息                            │
│   └─ 输出10列标准表格 → 用户确认或手动修正                   │
│   └─ 触发词: "supplier-ppt-extractor"                     │
└────────────────────┬────────────────────────────────────┘
                     ↓ [确认]
┌─────────────────────────────────────────────────────────┐
│ 节点3: 图片裁切（自动处理 → 人工确认/调整）                  │
│   └─ 裁切比例: 4:3                                        │
│   └─ 自动去除文案/水印（可选）                              │
│   └─ 显示预览 → 确认/调整/跳过                             │
│   └─ 部分图片需AI美化（人工标记）                           │
└────────────────────┬────────────────────────────────────┘
                     ↓ [确认]
┌─────────────────────────────────────────────────────────┐
│ 节点4: 飞书上传（自动上传 → 人工确认）                      │
│   └─ 输入: 飞书多维表格链接（用户粘贴）                      │
│   └─ 自动解析: base-token + table-id                      │
│   └─ 预设字段: 产品大类="礼盒礼包", 场景标签="端午节"        │
│   └─ 格式: batch-create {"fields": [...], "rows": [...]}  │
│   └─ 显示上传结果 → 确认/查看链接                           │
└────────────────────┬────────────────────────────────────┘
                     ↓ [确认]
              ┌──────────────┐
              │ 节点4.5: 数据获取 │
              │  （人工选择入口）  │ ← 可从飞书拉取已有数据复用
              └──────┬───────┘
                     ↓ [确认]
┌─────────────────────────────────────────────────────────┐
│ 节点5: 数据清理（自动清理 → 人工确认/调整规则）               │
│   └─ 显示清理结果 → 确认/调整规则                          │
│                                                         │
│   数据来源①: 节点4刚上传的新数据（新文件流程）               │
│   数据来源②: 从飞书多维表格拉取已有数据（复用已有数据）       │
│                                                         │
│   触发词: "数据清理"、"礼品数据处理"、"清洗Excel"            │
│   命令: python3 clean_data.py <输入.xlsx> [输出.xlsx]      │
└────────────────────┬────────────────────────────────────┘
                     ↓ [确认]
       ┌─────────────┐              ┌─────────────┐
       │ 节点6: 生成Excel│            │ 节点7: 生成PPT │
       │  （可单独触发）  │            │  （可单独触发）  │
       │  推品表格输出   │            │  占位符模板替换  │
       └──────┬──────┘              └──────┬──────┘
              ↓                            ↓
┌─────────────────────────────────────────────────────────┐
│ 节点8: 完成下载                                          │
│   └─ Excel / PPT 可单独下载                               │
│   └─ 额外: 精选产品升级方案（非标准化，人工挑选）              │
└─────────────────────────────────────────────────────────┘
```

### 二、节点2: 数据提取规则（10列标准输出）

#### 输出字段表

| #  | 字段名           | 内容要求                   | 示例                          |
| -- | ------------- | ---------------------- | --------------------------- |
| 1  | **编号**        | 产品编号/SKU               | `A001` / `SSP20260422283`   |
| 2  | **供应商名称**     | 来源PPT中的供应商             | `某某供应商`                     |
| 3  | **产品名称**      | **名称 - 型号**（短横线连接，无空格） | `松风雅韵-翻盖式礼盒配置A`             |
| 4  | **产品参数**      | 尺寸｜材质｜配件x数量（用 `｜` 分隔）  | `礼盒42x31x26cm｜纸盒｜茶杯x2｜茶宠x1` |
| 5  | **营销文案**      | 意境｜卖点（用 `｜` 分隔）        | `国潮设计｜手工粽+咸鸭蛋组合｜企业定制logo`   |
| 6  | **市价**        | 纯数字，无单位                | `299`                       |
| 7  | **供应商报价1**    | 纯数字，成本价                | `180`                       |
| 8  | **供应商一件代发报价** | 纯数字，代发成本价              | `195`                       |
| 9  | **备注**        | **仅报价条件**，不写型号/坐标      | `含税含运`                      |
| 10 | **京东/天猫链接**   | 完整URL                  | `https://...`               |

#### 强制规范（硬规则）

| 规范项    | 正确      | 错误                |
| ------ | ------- | ----------------- |
| 尺寸分隔符  | 小写 `x`  | 星号 `*`            |
| 配件数量符  | 小写 `x`  | 星号 `*`            |
| 产品名称格式 | `名称-型号` | `名称 型号` / 分开字段    |
| 备注内容   | 仅报价条件   | 型号、坐标、装箱数（除非特别要求） |
#### 列名智能匹配（容错）
| 目标字段   | 支持的输入列名                 |
| ------ | ----------------------- |
| 编号     | 商品编号、编号、Sku、SKU、商品编号-标品 |
| 供应商报价1 | 供货价、报价1、出货价、集采价、供应商报价1# |
| 营销文案   | 一句话卖点、卖点、一句话说明          |
| 市价     | 零售价、市场价、指导价             |

### 五、节点5: 数据清理规则
#### 输入 → 输出映射
| 输入列（飞书原始）          | 输出列（清洗后）    | 处理方式                     |
| ------------------ | ----------- | ------------------------ |
| 商品编号-标品 / 编号 / SKU | **编号**      | 原样保留                     |
| 供应商名称              | **供应商**     | 原样保留                     |
| 商品品名               | **名称**      | 原样保留                     |
| 产品参数               | **产品参数**    | 只保留组件列表，过滤基础包装           |
| 营销文案 / 一句话卖点       | **一句话卖点**   | `\n` 替换为 `｜`             |
| 市场价 / 零售价          | **市场价**     | 原样保留                     |
| 供应商报价1# / 供货价      | **供货价**     | 原样保留（成本价）                |
| 供货价（一件代发）          | **一件代发成本价** | 原样保留                     |
| -                  | **供货价报价**   | **自动计算**: 供货价 ÷ 0.75     |
| -                  | **一件代发报价**  | **自动计算**: 一件代发成本价 ÷ 0.75 |
#### 清理原则

1. **不要在原文件上直接修改** — 始终生成新文件
    
2. **空行自动跳过**
    
3. **找不到"供货价"列时报错退出**
    
4. **删除多余字段**，只保留10列标准输出

### 六、节点6: 生成Excel（推品表格）

#### 字段删减（客户不可见）
| 删除字段      | 原因       |
| --------- | -------- |
| 供应商名称     | 保护供应链信息  |
| 供货价（原成本价） | 成本价不给客户看 |
| 一件代发成本价   | 成本价不给客户看 |
#### 输出表头（10列）
| 列 | 字段名                | 来源     | 宽度 |
| - | ------------------ | ------ | -- |
| A | **编号**             | 编号     | 15 |
| B | **名称**             | 名称     | 25 |
| C | **产品参数**           | 产品参数   | 35 |
| D | **卖点简述**           | 一句话卖点  | 25 |
| E | **图片**             | 图片文件   | 12 |
| F | **市场价**            | 市场价    | 10 |
| G | **供货价（集采价，30套起订）** | 供货价报价  | 15 |
| H | **一件代发报价**         | 一件代发报价 | 12 |
| I | **备注**             | 保留空列   | 15 |
| J | **京东/天猫链接**        | 保留空列   | 20 |
#### 图片处理规则

- **缩放比例**: 宽高比自适应，最大 100×75 px
    
- **位置**: 每行对应图片放 E 列
    
- **映射**: 按行号一一对应，或按编号匹配
    

#### 行高设置

- 表头行: 25px
    
- 数据行: 80px（预留图片空间）

### 七、节点7: 生成PPT（占位符模板替换）

#### 核心流程
```
Excel数据 + 图片文件夹 → premium_ppt_placeholder.py → PPT文件
```
#### 模板占位符对照
| 占位符          | Excel列    | 说明         |
| ------------ | --------- | ---------- |
| `{{品名}}`     | 名称        | 产品名称       |
| `{{编号}}`     | 编号        | 商品编号       |
| `{{卖点摘要}}`   | 一句话卖点     | 营销文案       |
| `{{产品参数}}`   | 产品参数      | 规格参数       |
| `¥{{市场价}}`   | 市场价       | 零售价        |
| `¥{{供货价}}`   | **供货价报价** | ⚠️ 已加35%利润 |
| `¥{{一件代发价}}` | 一件代发报价    | 代发价        |
#### 图片匹配优先级

1. **按编号匹配**（优先）
    
    - 文件名格式: `{编号}.png` / `{编号}-{后缀}.png`
        
    - 示例: `SSP20260422283-TC01.png`、`SSP20260422284.png`
        
    - 自动提取编号部分匹配
        
2. **按页码匹配**（备选）
    
    - 文件名格式: `第1页.png`、`第2页.jpeg`
        
    - 按Excel行顺序对应: 行号+1 = 页码
        

#### 技术实现要点
| 项目   | 规则                                     |
| ---- | -------------------------------------- |
| 字体颜色 | 黑色 `#000000`                           |
| 字体样式 | 正体（非斜体）                                |
| 背景色  | 保持模板原样，不修改                             |
| 空值价格 | 显示 `¥-`（不能显示 `¥nan`）                   |
| 图片引用 | 每页独立图片文件 + 独立 `_rels` 配置               |
| 复制方式 | unpack → XML替换 → pack（不用python-pptx复制） |
#### 已知坑点 & 解决
| 坑点               | 现象           | 解决                                 |
| ---------------- | ------------ | ---------------------------------- |
| python-pptx复制不可靠 | 复制后slide保存失败 | 改用XML unpack/pack流程                |
| ¥nan问题           | 空价格显示`¥nan`  | 空值判断加 `!= 'nan'` 和 `strip() != ''` |
| 图片不显示            | 所有页显示同一张图    | grep确认模板rId，每页独立文件                 |
| 所有页同一张图          | rId冲突        | 每页独立图片文件名 + 独立\_rels               |
